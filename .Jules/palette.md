## 2025-02-18 - Improved Focus Visibility Across Core Form Elements
**Learning:** In complex data-entry interfaces like AAMVA field editing, `focus-visible` styles are critical. Elements like `<select>` and date-generation `<button>`s were easily lost during keyboard navigation because they matched the dark theme backgrounds or lacked clear borders on focus. Combining `focus-visible:ring-2` with `focus-visible:ring-brand-500` ensures a consistent and obvious visual indicator across different interactive elements (buttons, inputs, and selects), regardless of the theme (light/dark/DMV). Also, mapping `placeholder` text to dynamic properties like `field.dateFormat` drastically improves user expectation setting for strict format inputs without cluttering the UI with helper text.
**Action:** Always verify keyboard focus states (`tab` through the UI) for all interactive elements, especially custom inputs, and use dynamic placeholders driven by schema metadata where available to guide user input formats.
